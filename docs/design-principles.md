# Design Principles

## 1. Educational First

LOUIM exists to support teaching and learning.

Technical features are useful only if they help teachers reduce cognitive overload and introduce LibreOffice progressively.

## 2. Progressive Disclosure

Learners should see only the tools they need for the current learning objective.

More tools can be introduced later, in the order chosen by the teacher.

## 3. Teacher Empowerment

Teachers should be able to create, save, import, export, and share interface profiles without editing LibreOffice configuration files manually.

## 4. Language Independence

LOUIM must use LibreOffice UNO command IDs internally.

It must not depend on English, French, German, Italian, or any other localized menu labels.

## 5. Dynamic Discovery

LOUIM should discover the current LibreOffice interface whenever possible.

Hardcoded menu lists should only be used as a fallback.

## 6. Cross-platform Support

LOUIM must work on:

- Linux
- Windows
- macOS

The same template should work across all supported platforms.

## 7. Not a Lockdown Tool

LOUIM is not primarily designed to restrict users.

It is designed to simplify the interface for learning.

Administrative locking may be added later, but it is not part of the initial goal.
